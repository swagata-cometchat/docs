## Overview

CometChat Agent Builder gives product teams a guided flow to create AI copilots that feel native to their experience. Define your agent’s purpose, wire actions and tools, and publish to production UI (widgets or SDKs) without worrying about transport protocols or chat plumbing.

### What you get out of the box
- Built-in guardrails with conversation history, rate limits, and safe-tool invocation.
- UI previews for web widget and UI Kit surfaces while you iterate.
- Environment-aware tokens so stage and production apps can use the same agent safely.
- Analytics and debugging tools directly within the dashboard.

## Advanced

### Connect tools and MCPs together

When tools and MCP connections are both enabled, the agent can reason over structured data and trigger workflows in the same run. Use the <strong>Tool Invocation</strong> inspector inside the test console to confirm your schema keys match the payload your backend expects. If you are chaining MCP context with a tool call, pass identifiers (document IDs, meeting IDs, etc.) as part of the tool parameters so your service knows what the agent referenced.

### Tools

#### Overview

Tools expose deterministic skills to your agent—anything from “create a Zendesk ticket” to “fetch real-time order status.” Each tool defines its own JSON schema so the agent knows which fields it must collect before execution.

#### Custom API tools

1. Go to <strong>AI Agents → Tools → Create Tool</strong>.
2. Enter a display name that humans recognise (for example, <code>Zendesk Ticket Creator</code>).
3. Provide execution text that tells the agent when to call it: <code>Use this when a customer asks for a new support ticket.</code>
4. Define the tool name using lowercase-with-underscores. The name becomes the key in your action schema.
5. Describe the expected payload under <strong>Parameters</strong>. A typical schema looks like:

```json
{
  "type": "object",
  "properties": {
    "ticket_subject": {
      "type": "string",
      "description": "Short summary that will become the Zendesk subject"
    },
    "priority": {
      "type": "string",
      "enum": ["low", "normal", "high", "urgent"]
    }
  },
  "required": ["ticket_subject"]
}
```

6. Add authentication headers or secrets if your webhook requires them. Headers are encrypted at rest.
7. Connect the tool to an <strong>Action</strong> so the agent sees it. In your action schema, include a property whose key matches the tool name exactly.

<Callout className="mt-6" type="warning" title="Validate payloads">
  Run a few chats in the test console. Confirm the tool receives well-formed JSON before giving the agent production access to downstream APIs.
</Callout>

#### Google Suite tools

Google Suite tools bundle OAuth and service account flows so your agent can look up calendar events, draft emails, or read documents securely.

1. Create a Google Cloud project (or reuse an existing one) and enable the required APIs—Calendar, Gmail, Drive, etc.
2. Generate a service account with domain-wide delegation, then download the JSON key. Store it in a secret manager; you only need the file to copy values into the Agent Builder.
3. In <strong>AI Agents → Tools → Create Tool</strong>, choose <strong>Google Suite</strong> as the tool type.
4. Paste the service account email, client ID, and private key. Add the scopes your workflow needs, such as <code>https://www.googleapis.com/auth/calendar</code>.
5. Set the <strong>User to Impersonate</strong> (usually a shared inbox or administrator) so Google accepts requests.
6. Define the parameter schema that passes context to the tool. The builder populates common fields automatically, but you can extend it for custom use cases.
7. Save and test. The test console displays live Google API responses so you can confirm authorisation before launch.

### MCP

#### Overview

Model Context Protocol (MCP) connections let your agent pull just-in-time context—knowledge bases, product catalogue search, or any data source exposed over MCP. Each connection shows up as a context provider inside the builder, and responses are cached per run to stay within provider budgets.

#### Setup in Agent Builder

1. Navigate to <strong>AI Agents → MCP</strong> and click <strong>Connect MCP</strong>.
2. Supply the MCP server URL and label it so your teammates know what it surfaces (for example, <code>Support KB</code>).
3. Add authentication if the server requires API keys or Basic Auth.
4. Choose which agent(s) can access the connection. Stage and production apps can use separate secrets.
5. In the Agent Builder flow, open the <strong>Context</strong> panel and enable the MCP connection for the agent you are editing.
6. Use the test console’s <strong>Context Inspector</strong> to confirm the MCP server returns relevant documents.

<Callout className="mt-6" type="info" title="Chain MCP with tools">
  When an MCP snippet holds the identifier you need (for example, <code>order_id</code>), capture it in the tool parameters so the downstream action operates on the exact record the agent referenced.
</Callout>
